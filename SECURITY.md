# Security Policy

We take the security of RepoGuard, our multi-agent AI pipeline, and our users' data incredibly seriously. If you believe you have found a security vulnerability, please review the policy below to report it responsibly.

## Supported Versions

We actively monitor and patch the core multi-agent engine and dashboard layout. 

| Version | Supported          | Notes |
| ------- | ------------------ | ----- |
| 1.x.x   | :white_check_mark: | Active Stable Release Branch |
| < 1.0   | :x:                | Beta / Prototype branches (Deprecated) |

## Reporting a Vulnerability

**Please do not open a public GitHub Issue for security vulnerabilities.** Doing so exposes the exploit to malicious actors before a patch can be deployed.

Instead, please report security vulnerabilities via email to **aryanworks23@gmail.com**.

### What to Include in Your Report:
* **Description:** A detailed summary of the vulnerability.
* **Steps to Reproduce:** A clear, step-by-step proof of concept (PoC) or script so we can replicate the issue.
* **Impact:** What an attacker could achieve if this exploit is leveraged.

## Our Disclosure Process

Once a vulnerability report is received:
1. **Acknowledgement:** We will acknowledge receipt of your report within **48 hours**.
2. **Triage & Validation:** Our team will attempt to reproduce and validate the findings securely.
3. **Remediation:** If verified, we will work on a proactive patch. We aim to resolve validated high/critical vulnerabilities within **30 days**.
4. **Public Credit:** With your permission, we will happily credit you in our release notes for helping keep RepoGuard secure!
